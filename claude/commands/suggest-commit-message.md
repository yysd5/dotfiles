以下の手順で、ステージされた変更に対して適切なコミットメッセージを提案してください。

1. ステージされた変更の確認:
`git diff --staged` を実行して、現在ステージされている（git add済み）変更のみを確認してください。
**注意：勝手に `git add` を実行して未ステージのファイルをステージしたり、ステージされていない変更を含めてコミットメッセージを考えたりしないでください。** ステージされた変更がない場合は、その旨を伝えて終了してください。

2. git-committerスキルの確認:
`git-committer` スキルのガイドライン（50/72ルール、prefix、内容のガイドライン）に従ってください。

3. コミットメッセージの生成:
現在ステージされている差分とスキルのガイドラインに基づき、コミットメッセージを提案してください。
- サマリー（1行目）は現在形で記述し、日本語なら25文字以内を目指してください。
- 変更の「何」だけでなく「なぜ」に焦点を当て、必要に応じて詳細（3行目以降）を記述してください。
- 適切なprefix（feat, fix, refactorなど）を付与してください。
- ツール特有のメタデータ（"Generated with..." など）は含めないでください。

4. 提案の提示:
提案するコミットメッセージをコードブロックで提示し、その内容を選んだ理由を簡潔に説明してください。
「実際にコミットを行う場合は、上記のメッセージを使用して git commit を実行してください」と添えてください。
