# slack-info-getter

Slackから情報を得る時のガイドライン。
Slackから情報を得る際は、Slack MCPを使用する。

- 情報出力の際は、元となった投稿をユーザが自分でも読めるように、投稿のリンクも最後に注釈として記載するようにする
    - 日時、チャンネル、投稿者も併記する
- 同一スレッドの投稿については、スレッドの最初の投稿→`[1]`、そのスレッド内の特定の投稿→`[1-1]`のように、構造が分かるようにする
- 必ずしも出力した内容に結びつけないとリンクを載せてはいけないわけではなく、参考として何かのスレッドを載せても良い

以下が出力例

```
(略)

- xxxすることになった。[1]

(略)

- aaa作業ではkkkの問題が発生し、jjjを実施する方針に変更した。[2-1]

(略)

[1] [yyyについての議論スレッド上のxxxについての投稿(2026/3/2 11:00, #general, Aさん)](https://my-org.slack.com/xxxxxxxx)
[2] [aaaについての作業スレッド(2026/3/2 11:30, #random, Bさん)](https://my-org.slack.com/yyyyyyy)
[2-1] [aaa作業で発生した問題と方針転換についての投稿(2026/3/2 13:30, #random, Bさん)](https://my-org.slack.com/zzzzzzz)
```
